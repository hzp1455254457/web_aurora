# 项目文档目录

本目录存放项目相关的文档，按照类型分类组织。

## 📁 目录结构

```
docs/
├── design/          # 设计相关文档
│   └── figma-design-comparison.md  # Figma 设计对比检查
├── setup/           # 配置和设置文档
│   ├── figma-setup.md              # Figma 配置说明
│   ├── git-encoding-fix.md         # Git 编码修复指南
│   └── powershell-encoding-fix.md  # PowerShell 编码修复指南
└── guides/          # 使用指南
    ├── figma-mcp-guide.md          # Figma MCP 使用指南
    ├── figma-mcp-usage.md         # Figma MCP 详细使用说明
    ├── quick-start-git.md         # Git 快速开始指南
    └── view-page-guide.md         # 查看页面指南
```

## 📚 文档分类说明

### design/ - 设计文档
存放 UI/UX 设计相关的文档，包括设计对比、设计规范等。

### setup/ - 配置文档
存放项目配置、环境设置、工具配置等相关文档。

### guides/ - 使用指南
存放各种工具、功能的使用指南和快速开始文档。

## 🔗 相关文档

- **系统规范**：`openspec/specs/` - 系统规范定义
- **变更提案**：`openspec/changes/` - 变更提案和任务
- **项目说明**：根目录 `README.md` - 项目说明文档
- **OpenSpec 指南**：`openspec/AGENTS.md` - OpenSpec 使用指南

## 📝 文档规范

所有文档应遵循以下规范：
- 使用 Markdown 格式
- 文件名使用 kebab-case（小写字母和连字符）
- 文档开头包含简要说明
- 重要信息使用清晰的标题和列表
